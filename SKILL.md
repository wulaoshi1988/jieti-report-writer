# 课题研究报告写作（结题报告）

## 概述

将教育科研课题的结题材料撰写为规范的学术研究报告。核心原则：重点不是"做了哪些活动"，而是"为什么要研究、怎样开展研究、取得了什么成果"。默认使用学术表达、过去时/完成时，保证研究目标、内容、方法、成果一一对应。

## 适用范围

### ✅ 适用场景（触发本技能）

当用户说出以下内容时，应主动激活本技能：

- "帮我写结题报告""撰写研究报告""课题总报告怎么写"
- "课题要结题了，报告还没写""帮我整理结题材料"
- "研究报告的结构是什么样的""结题报告有哪些章节"
- 用户上传了课题相关材料（开题报告、中期报告、活动记录等），要求撰写结题报告

### ❌ 不适用场景（不要激活本技能）

以下情况请直接说明本技能不适用，不要强行激活：

- 用户需要写**开题报告**（本技能面向结题/研究报告，开题报告结构不同）
- 用户需要写**工作报告/活动总结**（本技能聚焦学术研究报告，非活动记录）
- 用户需要写**论文/期刊文章**（结构、篇幅、写作规范均不同）
- 用户仅需要**格式排版**（本技能负责内容写作，不负责 Word/PDF 排版）
- 课题不是教育科研类（本技能的模板和用语面向教育科研课题，其他领域可参考结构但需调整内容）

## 工作流程

### 第一步：信息采集

开始写作前，先向用户确认以下关键信息（如用户已提供则跳过对应问题）：

1. **课题基本信息**：课题名称、课题组、执笔人
2. **研究背景材料**：课题的研究背景（政策/学术/实践）、研究意义
3. **核心概念**：需要界定的术语、操作性定义
4. **研究设计**：研究目标（3-4个）、研究内容、研究方法（3-4种）
5. **研究过程**：阶段划分、时间节点、关键事件
6. **已有成果素材**：已发表的论文、形成的模式/策略、数据、案例等

**采集原则**：
- 一次性问 2-3 个问题，不要一次抛出所有问题
- 如果用户说"资料不全，先写吧"→ 进入第二步，在写作过程中注明 `[需补充：xxx]`
- 如果用户说"先不写了"/"暂停"→ 保存当前进度，告知用户随时可以说"继续写报告"恢复
- 必填项缺失时的处理：课题名称和研究目标是核心，缺失时暂停并提示用户补充；其他信息可在正文中标注 `[待补充]` 后继续

---

### 第二步：生成大纲

根据用户提供的信息，生成完整报告大纲。大纲需严格遵循模板的推荐结构，逐节列出各节的要点和规划篇幅。将大纲展示给用户确认后再进入逐节写作。

**特别注意**：
- "研究成果、主要结论与观点"应占总篇幅约 50%
- 确保研究目标、内容、方法、成果之间存在清晰的对应链条
- 如果用户提供的信息不足以生成完整大纲，在缺失处标注 `[需补充：xxx]`，并提示用户补充

---

### 第三步：逐节写作

按照大纲顺序逐节撰写。每写完一节，展示给用户审阅，确认后再进入下一节。

**写作过程中如遇到信息缺失**：
- 在该节对应位置标注 `[待补充：请用户提供xxx]`
- 告知用户需要提供什么信息，并继续下一节（不要让用户反复补充所有信息后再写）
- 示例提示语："这一节需要你提供研究过程中的关键事件，你可以简单描述几个重要节点，我来帮你整理成文字。"

**各节写作要点速查**（详细要求见 `references/研究报告写作模板.md`）：

| 章节 | 关键要点 |
|------|---------|
| 摘要 | 300字左右，重成果和结论，含关键数据 |
| 关键词 | 名词性术语，2-5个 |
| 研究背景 | 从政策/学术/实践中选，说明问题来源和必要性 |
| 研究意义 | 理论意义 + 实践意义 |
| 课题界定 | 核心概念的操作性定义 |
| 研究目标 | 动词+名词，3-4个 |
| 研究内容 | 与目标对应，有逻辑顺序 |
| 研究方法 | 3-4种，说明"做什么""怎么做" |
| 研究过程 | 分阶段，含时间/地点/对象/措施 |
| 研究成果 | 全文重点，论据+论证，忌只列论文/奖项；参考 `references/成果写作示范.md` |
| 研究效果 | 前后对比，量化优先 |
| 问题与设想 | 方法/样本局限，后续方向 |

---

### 第四步：质检自查

全文完成后，对照模板中的"自查清单"逐项检查，并输出检查结果：

- [ ] 是否清楚回答了"为什么研究、怎样研究、成果是什么"
- [ ] 是否把研究报告误写成工作报告
- [ ] 是否存在"流水账"写法
- [ ] 是否缺摘要、关键词、署名、参考文献、附录
- [ ] 研究目标、内容、方法、成果是否一一对应
- [ ] 是否有研究前后对比数据
- [ ] 是否存在大段复制、未标出处、整段雷同风险
- [ ] 是否还有 `[待补充]` 标记未处理

对未通过项给出具体修改建议。

## 常见问题（FAQ）

**Q：我的课题资料很少，能写吗？**
A：可以。先根据已有信息写出初稿，缺失的部分用 `[待补充]` 标注，你后续补充后再完善。不要等所有资料齐全再开始。

**Q：研究报告和工作报告有什么区别？**
A：工作报告写"开展了什么活动"（过程记录），研究报告写"研究发现了什么"（成果和结论）。本技能帮您写的是后者。

**Q：研究成果那一节不知道怎么写？**
A：这是最常见的问题。请参考 `references/成果写作示范.md`，里面有错误写法和正确写法的完整对比，以及不同类型成果的写作模板。

**Q：写到一半想暂停，怎么办？**
A：直接说"暂停"或"先到这里"，我会保存当前进度。之后说"继续写报告"即可从中断处恢复。

**Q：结题报告一般多少字？**
A：教育科研课题的结题报告一般在 8000-15000 字之间，具体以课题管理部门的要求为准。"研究成果"章节应占约 50%。

**Q：需要参考文献吗？**
A：需要。请列出研究过程中真正参考过的重要文献，格式参考 GB/T 7714-2015。如没有提供参考文献，我会在自查环节提醒。

## 注意事项

- **学术语气**：全文使用学术表达，避免口语化、抒情化
- **时态一致**：默认使用过去时和完成时
- **区分报告类型**：研究报告 ≠ 工作报告，重点在"研究发现了什么"而非"开展了什么活动"
- **成果写作禁忌**：不要只列论文题目、获奖名称、活动次数——这些都是成果形式，不是成果本身。要写"数据 + 典型事例 + 具体描述"；详见 `references/成果写作示范.md`
- **文献引用**：引用他人观点/数据时使用脚注，参考文献格式参考 GB/T 7714-2015
- **附录原则**：少而精，只放对正文有支撑价值的材料
- **处理 `[待补充]` 标记**：全文完成后，统一整理所有待补充项，一次性向用户确认
